## Verdict and scores

"A unique synthesis, rather than an invention composed entirely of new
technical primitives." Concept 9/10 · original synthesis 8.5 · technical
proof 7.5 · **visual proof 6.5** · potential 9 · clarity 9.

## What the audit confirms (with better words than ours)

- **The distilled thesis:** "The emotional effect of a diffusion-rendered
  interface can be separated from the expensive act of generating every
  pixel." → adopted framing for [[thesis]].
- **The better problem:** generative UI asks how a model constructs the
  right interface; painted UI asks what makes it *feel alive* — a motion/
  choreography layer that sits *on top of* protocols like A2UI, not a
  competitor. "Probably the most defensible original contribution."
- **Originality ratings:** the individually-established parts rate Low/
  Established; the High-originality items are exactly the synthesis:
  *streaming patches as visible painting rhythm; reproducing video-gen
  aesthetics without video; the complete framing and named technique.*
- Endorses the four-behavior decomposition ([[concepts/the-illusion]]) as
  "an engineering hypothesis," the four-loop implementation
  ([[concepts/four-loops]]), "token pacing becomes choreography"
  ([[concepts/latency-and-streaming]]), the failure logs, and the
  [[concepts/choreographer]] as "commercially valuable."

## What the audit corrects (the work list)

1. **⚠️ The visual gap is the whole game now (6.5/10).** "Current scenes
   can still resemble animated dashboards… rather than something truly
   painted." Hands-on confirmation on exp-08: reviewer "cannot tell the
   difference"; resizing the browser "did not do anything special" —
   exp-08's harness omitted spring reflow (ingredient 1) entirely.
   exp-08 is **paused**; the demo must get wonderful before perception
   testing is meaningful. → exp-09 showcase.
2. **Richer primitives needed:** paths, masks, blur, brush/texture
   shaders, compound components, shared-element identity, layered cached
   rasters → extends [[concepts/experience-frontier]] lever 1.
3. **Constraint layout** (alignment, intrinsic sizing, wrapping, safe
   areas, collision) → [[concepts/fluid-layout]] open questions, now with
   a concrete requirements list.
4. **Accessibility mirror as a first-class loop:** nodes already carry
   `semantic` — generate a synchronized hidden DOM accessibility tree.
   New planned page: `concepts/accessibility-mirror`.
5. **Key handling:** label exp-03 live mode demo-only in README (done);
   deployed versions need a server-side relay.
6. **The decisive evaluation** must include an actual video-diffusion
   interface as a condition, and painted UI must be tested *with strong
   assets and shaders* — a weak painted condition tests nothing (exp-08's
   lesson).

## The reframed opportunity

Not "a cheaper Flipbook" but a **runtime for cinematic agent interfaces**:
agents send semantic intent; painted UI is the experiential layer that
answers *"once the agent has declared the interface, how should it come
alive?"* — org-supplied primitive catalogs and motion grammars, local
choreographer enforcing brand + accessibility, cloud model for reasoning,
local SLM for rapid visual reaction. This is the product framing to carry
forward.
