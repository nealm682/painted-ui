# Overview — Painted UI

This wiki is about one question: how do you make a screen *feel* like it's
being painted live by a model — the Flipbook illusion — without paying
Flipbook's cost?

## The reference point: Flipbook

Flipbook (Zain Shah, Eddie Jiao, Drew Carr; launched Apr 2026) streams live
1080p video at 24fps generated by a heavily-optimized version of LTX Studio's
video-diffusion model, over websockets, on Modal serverless GPUs. There is no
HTML and no layout engine — every pixel is generated. The result is striking:
illustrations reshape to fit the window, and any region can become interactive.
The team openly notes it's early, slow, that demos were sped up/edited, and
that they hit capacity/waiting-room problems. See [[entities/flipbook]].

## The problem

Real-time video diffusion has broken unit economics for a product: you rent a
high-end GPU per active user per second, with no cross-user caching and cost
scaling linearly with concurrency. Great demo, hard business. See
[[concepts/video-diffusion-approach]] and [[concepts/cost-model]].

## The bet

The "wow" is a *behavior*, not a rendering method. If you can reproduce that
behavior with a model that authors a resolution-independent scene, rendered by
a compositor on the user's own GPU, you get the same illusion at near-zero
marginal cost. See [[thesis]] and [[concepts/the-illusion]].

## Where this is heading

Two research fronts are converging toward this: (1) diffusion is getting fast
enough to stream in near-real-time via consistency/distillation methods, and
(2) "generative UI" is maturing into models authoring structured, interactive
interfaces at runtime. This wiki tracks both and synthesizes the cheap path
between them.
