The architecture

Four roles. One shared stage.

The architecture works because intelligence, motion policy, state, and rendering are separated. Each operates at its own speed without blocking the others.

01 · LLM / agent

Director

Decides what exists and what changes. Emits semantic operations instead of pixels or animation timelines.

02 · local policy

Choreographer

Decides how changes move: verb, duration, easing, direction, and stagger. Deterministic and brand-governed.

03 · node map

Stage

Holds persistent semantic objects. Stable IDs preserve identity, enabling morphing, continuity, and interruption handling.

04 · compositor

Painter

Reads current state and redraws continuously. It executes approved verbs at display speed and never waits for the network.

Enterprise value

Generative UI with a governance boundary.

The model controls meaning and intent. The product team controls visual language, motion, accessibility policy, and rendering code.

Model micromanages the UI

High expressiveness, but every detail becomes a probabilistic output.

  • More tokens and latency
  • Inconsistent brand behavior
  • Difficult to unit test
  • Unsafe to allow arbitrary code
  • Transitions may conflict or snap

Model directs a governed runtime

Meaning stays generative while execution remains deterministic.

  • Small allow-listed protocol
  • Versioned motion design system
  • Replayable and auditable patches
  • Client-side performance
  • Optional model overrides within policy
The honest claim

A distinctive synthesis—not magic.

“Traditional design systems govern what components look like. Painted UI adds a governed vocabulary for how generated experiences come into existence and change over time.”The contribution is the composition: semantic streaming, persistent identity, local choreography, and continuous rendering.
Established ingredients: scene graphs, tweening, streaming protocols, game loops, and immediate-mode rendering.
Distinctive synthesis: using those ingredients as a semantic motion runtime for model-authored interfaces.
Not yet proven: broad enterprise task coverage, accessibility, measurable usability gains, and superiority to component UI.
Likely production form: a hybrid—standard accessible components plus Painted UI for focus, continuity, explanation, and transitions.
Recommended next step

Prove it on one real workflow.

Generate a result set, focus one item, morph it into detail, complete an action, then return without losing spatial context. Measure task completion, perceived latency, comprehension, and preference against a conventional generative UI.
