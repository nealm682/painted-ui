The key idea: the render loop (4) never waits for the network (2). Streaming mutates state; rendering samples state. A node mounted mid-stream simply starts existing — and its enter animation plays while later tokens are still arriving. That overlap is the entire "painted live" effect: token pacing becomes choreography.

The four loops in the real code

Loop 1 — the agentic conversation. Each Paint or click pushes a user message and calls the model once. Clicks send {"event":"click","semantic":…}, so the model sees its own scene history and replies with a short mutation — not a repaint. 
messages.push({role:"user",content:userContent});   // your turn
… stream runs …
messages.push({role:"assistant",content:assistantAcc}); // its turn, remembered
Loop 2 — the network read. Yes: this is the streaming loop you asked about. fetch returns a byte stream; we loop on await reader.read(). Each wake-up delivers a chunk of Server-Sent Events; each content_block_delta inside holds a few characters of the model's text. Chunks split anywhere — mid-word, mid-number. 
while(true){ const {done,value}=await reader.read(); if(done)break;
  …parse SSE events…  feed(delta.text); }   // feed() gets arbitrary fragments
Loop 3 — the parser scan. feed() appends the fragment to a buffer and walks it character by character, tracking depth (brace nesting), inStr (inside quotes — braces in strings don't count), and esc (backslash). When depth returns to 0, one complete {…} exists → JSON.parse → applyPatch. Incomplete tails wait in the buffer for the next chunk. This is why pretty-printing, fences, and prose can't break it — and the sanitizer fallback forgives .5-style decimals (run 2's lesson).
Loop 4 — the render loop. requestAnimationFrame(draw) runs 60×/sec no matter what the network does. It iterates the node map, asks each tween for its current value (cur(n,"x")), and draws. Enter/exit/morph animations are just functions of time since the patch arrived — no timeline, no queue, no coordination with the stream at all.

Why this feels like painting

A web page loads, then shows. A video streams pixels that are already composed. This does neither: structure arrives in reading order at token speed, each piece animates in as it lands, ambient motion never stops, and the next interaction mutates the living scene instead of replacing it. The model is the director; your GPU is the painter; the stream is the brush moving across the canvas.
